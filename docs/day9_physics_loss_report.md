# Day 9: Physics Loss Implementation

## Project Title
Benchmarking Physics-Informed Neural Networks (PINNs) and Fourier Neural Operators (FNOs) for Aerodynamic Surrogate Modeling Using the NASA Airfoil Learning Dataset

---

## 1. Objective

Implement a physics loss module (`physics_loss.py`) that encodes aerodynamic physical constraints as differentiable penalty terms, to be added on top of the data-driven loss during PINN training (Day 10).

---

## 2. Deliverable

**`physics_loss.py`** — a standalone, importable Python module containing:
- `PhysicsLoss` — a `torch.nn.Module` implementing four aerodynamic constraint penalties
- `verify_constraints()` — a diagnostic function that measures constraint satisfaction rates on any set of model predictions

---

## 3. Clarification: This is Not a Classical PINN

The terminology "PINN" is used loosely in this project. A strict clarification is necessary before reporting results.

A **classical PINN** computes PDE residuals — it plugs the network's output into a governing differential equation (e.g. Navier-Stokes, continuity) at collocation points in the flow domain and penalizes violations:

```
residual = ∂u/∂x + ∂v/∂y   # continuity — should be zero everywhere
```

This requires velocity and pressure fields on a spatial mesh, and automatic differentiation with respect to spatial coordinates.

**This project cannot implement classical PDE residuals** because the NASA Airfoil Learning Dataset does not contain:
- Velocity fields (u, v)
- Pressure fields on a CFD mesh
- Spatial coordinates of the flow domain

What `physics_loss.py` implements instead are **soft aerodynamic constraint penalties** — known physical relationships and physical bounds expressed as differentiable loss terms. This is more accurately described as a **physics-constrained neural network** rather than a full PINN. This distinction should be stated clearly in the final paper.

---

## 4. Automatic Differentiation

`physics_loss.py` does not implement automatic differentiation — it relies entirely on PyTorch's built-in autograd engine.

Every operation inside `PhysicsLoss.forward()` uses standard PyTorch tensor operations (`nn.functional.mse_loss`, `torch.relu`, `.sum()`, `.mean()`, tensor slicing). Because the model predictions passed in are themselves outputs of a PyTorch model, the computation graph is already being tracked by autograd. Calling `.backward()` on the total loss automatically propagates gradients through every physics term back into the model weights.

No extra implementation is needed. PyTorch handles this transparently.

---

## 5. Physics Constraints Implemented

Four constraints are implemented, chosen specifically because they are enforceable from the quantities available in this dataset (surface Cp distribution + scalar aerodynamic coefficients).

---

### 5.1 Lift-Pressure Consistency

**Physical basis:** Lift is generated by the pressure difference between the upper and lower airfoil surfaces. The lift coefficient Cl is proportional to the chord-wise integral of the Cp distribution:

```
Cl ∝ ∫ Cp dx
```

**Implementation:** The integral is approximated using the trapezoidal rule on the 98-point uniform chord grid. The resulting estimated Cl is compared against the network's directly predicted Cl using MSE:

```python
dx = 1.0 / (n_cp - 1)
cl_from_cp = 0.5 * (pred_cp[:, :-1] + pred_cp[:, 1:]).sum(dim=1) * dx
L_lift = MSE(pred_cl, cl_from_cp)
```

**What this enforces:** The model's Cp prediction and its Cl prediction must be internally consistent. Without this, the network can predict a plausible-looking Cp curve and a plausible Cl value that have nothing to do with each other.

**Weight:** λ₁ = 1.0

---

### 5.2 Pressure Smoothness

**Physical basis:** Pressure distributions around aerodynamic surfaces are physically smooth except near sharp flow transitions (e.g. shock waves, leading-edge separation). High-frequency oscillations in predicted Cp are non-physical.

**Implementation:** Penalizes the mean squared difference between adjacent Cp points:

```python
diff = pred_cp[:, 1:] - pred_cp[:, :-1]
L_smooth = (diff ** 2).mean()
```

**What this enforces:** Smooth, physically realistic pressure distributions. Acts as a spatial regularizer on the Cp output.

**Weight:** λ₂ = 0.01 (small, since some variation between adjacent points is physically valid)

---

### 5.3 Drag Positivity

**Physical basis:** Aerodynamic drag cannot be negative under normal operating conditions.

**Implementation:** Uses ReLU to penalize only negative Cd predictions, with zero penalty when Cd ≥ 0:

```python
L_drag = ReLU(-pred_cd).mean()
```

**What this enforces:** Non-negative drag predictions. The penalty is zero for valid predictions and grows linearly with the magnitude of violation.

**Weight:** λ₃ = 0.1

---

### 5.4 Drag Consistency

**Physical basis:** Total drag (Cd) is the sum of pressure drag (Cdp) and skin-friction drag. Therefore Cd ≥ Cdp must always hold — total drag cannot be less than one of its components.

**Implementation:** Uses ground-truth Cdp labels (available in the dataset) as the reference, since Cdp is not a model output:

```python
L_cdp = ReLU(true_cdp - pred_cd).mean()
```

**What this enforces:** The model's Cd prediction must be at least as large as the known pressure drag component. Set `lambda_cdp=0` if Cdp is not available in your target tensor.

**Weight:** λ₄ = 0.1

---

## 6. What Is Not Computed (and Why)

| Constraint | Why absent |
|---|---|
| Navier-Stokes residuals | Requires velocity/pressure fields on a mesh — not in dataset |
| Continuity equation | Same reason |
| Moment consistency (Cm ∝ ∫ Cp·x dx) | Cm is not a model output in the current architecture; add if Cm prediction is added |
| Cp physical bounds | Bounds depend on Mach number and are in physical units; original scalers not available to map bounds to normalized space |

---

## 7. Total Loss Formulation

During PINN training (Day 10), the total loss combines the weighted data loss from Day 6 with the physics penalty:

```
L_total = L_data + λ₁·L_lift + λ₂·L_smooth + λ₃·L_drag + λ₄·L_cdp
```

where `L_data` is the weighted MSE from Day 6:

```
L_data = 49.0 · MSE(Cl, Cd) + 1.0 · MSE(Cp)
```

Initial physics weights (to be tuned in Day 11):

| Weight | Value |
|--------|-------|
| λ₁ (lift consistency) | 1.0 |
| λ₂ (smoothness) | 0.01 |
| λ₃ (drag positivity) | 0.1 |
| λ₄ (drag consistency) | 0.1 |

---

## 8. verify_constraints()

A separate diagnostic function that measures physical constraint satisfaction on any set of predictions — not a loss, just a measurement.

| Output | What it measures |
|---|---|
| `drag_positivity_rate` | Fraction of predictions where Cd ≥ 0 (0.0 to 1.0) |
| `drag_consistency_rate` | Fraction where Cd ≥ Cdp ground truth (0.0 to 1.0) |
| `mean_cp_smoothness` | Mean squared adjacent Cp difference — lower is smoother |
| `lift_cp_correlation` | Pearson correlation between pred Cl and ∫pred Cp dx — higher means better internal consistency |

The `verbose` parameter controls whether results are printed to console (`True`, default) or returned silently as a dict (`False`). Use `verbose=False` inside training loops to avoid cluttering output while still logging values.

Call this after training to confirm the physics losses are actually improving physical consistency, not just reducing loss numbers.

## 9. Usage

```python
from physics_loss import PhysicsLoss, verify_constraints

physics = PhysicsLoss(
    lambda_lift=1.0,
    lambda_smooth=0.01,
    lambda_drag=0.1,
    lambda_cdp=0.1
)

# Inside training loop
pred = model(x)
data_loss = weighted_loss(pred, y)
physics_losses = physics(pred, y)
total_loss = data_loss + physics_losses['total']
total_loss.backward()

# After training
with torch.no_grad():
    pred = model(X_val_t)
verify_constraints(pred, y_val_t)
```

## 10. Limitations

- **Not a classical PINN.** No PDE residuals are computed. The governing Navier-Stokes equations are not enforced at any point. Results should be framed as physics-constrained surrogate modeling in the paper.
- **Lift consistency is approximate.** The trapezoidal integration on a normalized chord grid is a surrogate for the true aerodynamic lift integral, which involves the actual chord length and pressure difference between suction and pressure surfaces. The proportionality constant is absorbed implicitly by the MSE minimization.
- **Cdp uses ground-truth labels.** The drag consistency constraint cannot be applied at inference time on new data where Cdp is unknown.
- **Weights are empirical.** λ₁–λ₄ are initial values from the Day 8 formulation and will be tuned experimentally in Day 11.
- **Dataset is 2D.** All constraints are derived from 2D airfoil aerodynamics. 3D wing effects are not captured.
